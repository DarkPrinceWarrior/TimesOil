# Аудит покрытия датасета и план дальнейших улучшений

Дата: 2026-07-07. Основание: журнал проекта + четыре целевые разведки
(дообучение предобученных моделей; непокрытые приёмы для заводнения;
обход библиотек временных рядов; физика межскважинного взаимодействия
2025–2026). Все утверждения разведок подтверждены первоисточниками.

## 1. Что уже выжато из датасета (аудит покрытия)

**Сопутствующие ряды и признаки — покрыты:**
закачка (блочные суммы; адресная по гидропроводности $\bar k/d^2$; по
связности CRM), ряд ёмкостно-резистивной модели как известный наперёд,
пластовое давление (историческая ковариата), статика (проницаемость,
пористость, толщина) и блок разлома (в обучаемых моделях), координаты
(через веса аллокации), лаги и скользящие цели, наработка часов (на
Volve). Сознательно не использованы: давление нагнетания (дублирует
закачку), забойное давление добывающих (константа 60 атм — ноль
информации), обводнённость как отдельная ковариата (входит через ряд CRM
и фракционную модель).

**Модели — 16+ семейств:** наивный, экспонента, Арпс; CRM, CRM-P,
фракционная модель Джентила; TiRex-2 (8 конфигураций), Chronos-2 (4),
SPDM, LightGBM, сети BiTCN/NHITS/TiDE; ансамбли (среднее топ-2, обратные
ошибки, NNLS). Проверены и мотивированно отклонены: Toto-2 (нет
ковариат), Timer-S1 (нет ковариат, 40 ГБ), Falcon-X, TimeRouter,
Moirai-2 и TabPFN-веса (некоммерческие лицензии), TTM (не месячные
частоты), мёртвые библиотеки (Merlion, Kats).

**Методики:** скользящая проверка 3 и 14 срезов без утечки, конформная
калибровка, ансамбль с честными весами, оптимизация закачки, проверка
переносимости на двух внешних полигонах.

**Достигнутый уровень:** нефть 5.10 % WAPE, жидкость 3.87 % (ансамбль,
канонические срезы). Смещения ≤ 2 %.

**Вердикт по «максимуму»:** по классу «модели в лоб» — да, практический
максимум достигнут: все живые семейства моделей с поддержкой наших
ковариат проверены, и дальнейший перебор моделей ничего не даст.
Непокрытыми остались рычаги трёх видов: (а) более глубокая физика,
использующая наш уникальный ресурс — помесячное пластовое давление;
(б) обученный стекинг вместо линейных весов; (в) точечные инструменты
(строгая калибровка, признаки смены режима). Итого потенциал ещё
~0.3–1 п.п. WAPE по нефти реалистичен, дальше — шум данных.

## 2. Ключевые находки разведок

### 2.1. Физика межскважинного взаимодействия 2025–2026 (главное)

1. **CRMP-TM (тюменская школа: Степанов, Бекман и др.; ПРОНЕФТЬ
   2025;10(3):44–59, открытый доступ)** — семейство CRM, где
   **пластовое давление — данные адаптации**, а не побочная колонка:
   восстанавливается динамика давления вокруг скважин, целевая функция
   настраивается на разности пластовых давлений; попутно — оценка
   непроизводительной закачки (~0.5 %) и модель обводнённости,
   согласованная с CRMP. У нас помесячное пластовое давление по каждой
   добывающей — редкая роскошь (обычно 1–2 замера в год), и сейчас оно
   используется только как ковариата. **Единственная найденная методика,
   превращающая этот ресурс в физику.** Кода нет, уравнения опубликованы
   полностью; реализация поверх нашего `crm.py` — 1–2 недели.
2. **Двухфазная CRM с переменной продуктивностью** (Zhanabayeva,
   Pourafshary, Scientific Reports 2026; русский аналог — Галеев,
   Синицына, Вестник ТюмГУ 2025, 11(4):70–92, открытый PDF):
   насыщенность из материального баланса нефти, продуктивность и
   $\tau(t)$ обновляются по фазовым проницаемостям — **жидкость и нефть
   согласованно внутри одной модели**, без стыка «CRM × Джентил».
   На полевом кейсе вдвое ниже ошибка классической CRM. Реализация —
   ~1 неделя (формулы полные).
3. **Вейвлет-когерентность закачка↔добыча** (Gabry et al., Energies
   2026, 19(9):2211) — независимая диагностика связности без трассеров:
   маска-регуляризатор для $f_{ij}$ (обнуление некогерентных пар),
   проверка экранов разломов. 1–2 дня.
4. Отклонено: INSIM-3P (SPE 2025 — сильная физика, но код закрыт,
   3+ недели самописной реализации), multiwell deconvolution (нужны
   непрерывные записи давления), state-space/Калман (свежих методик под
   месячные ряды нет).

### 2.2. Дообучение предобученных моделей

Официальное дообучение есть у Chronos-2 (LoRA через `fit()` или
AutoGluon `fine_tune: True`), TimesFM-2.5 и Toto; **у TiRex-2 — нет**
(только коммерческая услуга NX-AI). Но: рекомендация AutoGluon — от 100
рядов (у нас 33); литература на малых панелях даёт −5…+5 % с медианой
около нуля; LoRA безопаснее полного дообучения. Вердикт: один
контролируемый вечерний эксперимент Chronos-2 LoRA, не более; при
деградации — откат.

### 2.3. Непокрытые приёмы

1. **Обученный стекинг второго уровня вместо NNLS** (AutoML Conf 2025:
   систематически лучше линейных комбинаций; реализации — AutoGluon
   `PerItemGreedyEnsemble`, веса «модель × горизонт»): ожидание
   0.2–1 п.п. WAPE, 1–2 дня.
2. **TabPFN-TS** — документированно сильна ровно в нашем режиме
   (короткие ряды + сильные ковариаты; на ковариатных задачах обходит
   TiRex/Chronos-Bolt). **Ограничение: веса под некоммерческой
   лицензией** — допустима только как исследовательское сравнение, в
   продуктив не берётся.
3. Давление как вспомогательная цель / материальный баланс как
   регуляризатор обучаемых моделей — доли процента, низкий риск.
4. Копула по остаткам (согласованные сценарии нефть/жидкость) и
   иерархический байес для параметров Джентила — для интервалов, не для
   точечного WAPE.

### 2.4. Библиотеки

Брать точечно: **crepes** (конформная калибровка на голых остатках,
мондрианова таксономия «горизонт × блок», формальные проверки
применимости), **ruptures** (признаки смены режима закачки для
LightGBM), **AutoGluon-TimeSeries** (контур дообучения Chronos-2 и
независимая проверка ансамбля), pycatch22 (мета-признаки, только
диагностика). Не брать: sktime/aeon/GluonTS/pytorch-forecasting/
skforecast (дублируют написанное), MAPIE (не ложится на панель и
гетерогенный стек), Merlion/Kats (мертвы).

### Волна 3 — опционально

Давление как вспомогательная цель в обучаемых моделях; копула по

### Не делать (обоснованно)

точечная модель, претрейн на аргентинском корпусе (несовпадение режимов),
дообучение TiRex-2 (недоступно), TTM/Toto (чужие частоты/домен),
переезд на sktime/GluonTS (churn без выигрыша).

## 4. Ожидаемый потолок

Ансамбль 5.10 % (нефть) / 3.87 % (жидкость) → после волн 1–2 реалистично
**4.3–4.8 % / 3.4–3.7 %**. Дальше — неустранимый предел: горизонт 6
месяцев, режимные изменения закачки в контрольных окнах и дискретизация
до месяца. Для практических задач (планирование добычи, перераспределение
закачки) текущая точность уже избыточна относительно типовой промысловой
погрешности замеров.

«двухфазная CRM + дообученный Chronos-2 + обученный стекинг».

финальным стеком (требуются режимы «вперёд» для двухфазной CRM и
дообученного Chronos-2) и обновление PDF-приложения.
