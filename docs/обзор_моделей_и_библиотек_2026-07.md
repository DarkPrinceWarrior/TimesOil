# Обзор моделей и библиотек прогнозирования временных рядов (июль 2026)

Дата: 2026-07-07. Исполнитель: агент TimesOil.
Повод: внешний обзор предобученных моделей (Toto 2.0, TiRex-2, Timer-S1,
Falcon-X, TimeRouter, SPDM, MS-FLOW) и предложение изучить библиотеки
darts и neuralforecast.
Метод проверки: карточки HuggingFace (через API), репозитории GitHub
(клонирование, чтение исходников), статьи arXiv, **живые** таблицы
лидеров GIFT-Eval и fev-bench (срез 2026-07-07), установка и смоук-тесты
darts 0.45.0 и neuralforecast 3.1.9.

Контекст примерки: 33 месячных ряда × ~100 точек, горизонт 6 месяцев;
критичны известные наперёд сопутствующие ряды (закачка — управляющий
план) и исторические (давления); нужны квантили; CPU или A100-40GB;
коммерческая лицензия. Достигнутые результаты (WAPE, 3 среза × 6 мес):
CRM × Джентил — нефть 6.4 %, CRM — жидкость 4.4 %; TiRex-2 с ковариатами
7.8 % / 4.8 %; без ковариат любой метод ≥ 9.1 % / 13.4 %.

## 1. Проверка внешнего обзора по фактам

Стратегический тезис обзора — «не искать одну модель, сравнивать всё на
едином скользящем бэктесте» — совпадает с нашей методикой и уже
подтверждён тремя полигонами (§ 7 основного отчёта). Фактические
поправки:

### 1.1. Toto 2.0 (Datadog) — первое место обзора не подтверждается

- **Сопутствующих рядов сейчас нет вообще.** Цитата из репозитория
  DataDog/toto: «Fine-tuning and exogenous variable (EV) support are
  planned for a future 2.0 release but not yet available. If you need
  these features today, use Toto 1.0»; в Toto 1.0 экзогенные переменные —
  только через дообучение, не в режиме без обучения.
- **На живой таблице GIFT-Eval TiRex-2 стоит выше**: TiRex-2 — № 17
  (ранги MASE/CRPS 29.08/27.60), Toto-2.0-2.5B — № 19 (29.61/29.28).
- **Обучающие данные** — только синтетика и телеметрия наблюдаемости
  (цитата техотчёта: «…despite training only on synthetic and
  observability data»); реальных месячных промысловых рядов в обучении
  нет — риск домена для нашей задачи.
- Наши абляции показывают цену отсутствия закачки: потолок без
  сопутствующих рядов — 9.1 % (нефть) и 13.4 % (жидкость, хуже наивного).
- Вердикт: **не кандидат в лидеры**; полезен как «верхняя планка прогноза
  без сопутствующих рядов» (достаточно версии 313m, 1.2 ГБ, Apache-2.0).

### 1.2. Главный пропуск обзора — Chronos-2 (Amazon)

Единственная зрелая модель, покрывающая наш сценарий наравне с TiRex-2:

- нативные исторические и известные наперёд сопутствующие ряды
  (числовые и категориальные) — цитата карточки: «Chronos-2 supports all
  covariate types natively»;
- произвольные уровни квантилей; контекст до 8192; 120 млн параметров;
  инференс на CPU («300+ прогнозов в секунду на A10G»); Apache-2.0;
- табличный API: `Chronos2Pipeline.predict_df(context_df, future_df=…)` —
  план закачки подаётся напрямую;
- **№ 1 на fev-bench** (Skill Score 35.50, Win Rate 88.07) — единственном
  бенчмарке, проверяющем именно работу с сопутствующими рядами; на
  GIFT-Eval — № 27 (ниже TiRex-2).

Вердикт: **обязателен к прогону на всех трёх полигонах** — прямой
конкурент TiRex-2 и партнёр по ансамблю.

## 2. Библиотека darts (unit8co), версия 0.45.0

Python 3.13 официально не поддержан; работает с пином `numba>=0.60`
(проверено установкой и смоук-тестом «33 скважины × 100 точек, горизонт
6, 3 среза»).

**Брать:**

1. **`Chronos2Model`** — единственная предобученная модель в darts,
   не теряющая сопутствующие ряды; бонус — объяснимость прогноза
   (`ShapExplainer`).
2. **Бэктест-контур**: `historical_forecasts` / `backtest` с
   `retrain=False` — расширение с 3 до 10–20 срезов почти бесплатно;
   готовые метрики `wmape` (= наш WAPE), `merr` (= смещение) и
   интервальные `mic`/`miw` (накрытие/ширина); прогнозы CRM подаются в
   `backtest(historical_forecasts=…)` без обёрток — единая таблица
   «физика / предобученные / обучаемые / классика».
3. **`LightGBMModel`/`CatBoostModel`** с `lags_future_covariates`
   (закачка на шаги 1–6), статической ковариатой «блок разлома»,
   `likelihood="quantile"` + **`ConformalNaiveModel`/`ConformalQRModel`**
   (погоризонтная конформная калибровка из коробки, аддитивная —
   независимая проверка нашей мультипликативной).

**Ограничения (проверено по коду):** обёртка TiRex в darts — это
**TiRex-1 без сопутствующих рядов** (`supports_future_covariates ->
False`), наш раннер TiRex-2 незаменим; TimesFM-2.5 в darts — без
экзогенной регрессии; физики (CRM, обводнённость) в darts нет; из
классики полезны AutoARIMA/AutoETS (принимают закачку как регрессор),
AutoTheta, TBATS.

## 3. Экосистема Nixtla

Проверено по исходникам (neuralforecast 3.1.9, statsforecast 2.0.3,
mlforecast 1.0.31, hierarchicalforecast 1.5.1; Python 3.13 — да;
внимание: main neuralforecast тянет torch>=2.9.1 и ray/optuna — ставить
отдельным окружением от TiRex-2 с torch<2.10).

**Брать:**

1. **mlforecast + LightGBM** — лучший шанс среди обучаемых (~40–50 %
   выйти в 7–9 % на нашем поле): закачка честно через `X_df`
   (известные наперёд значения), давления — лагами ≥ горизонта, статика
   и блок — `static_features`, конформные интервалы и кросс-валидация из
   коробки. Трудоёмкость ~1 день.
2. **statsforecast**: AutoARIMA (умеет закачку как регрессор), AutoETS,
   AutoTheta — классические эталоны, считаются секунды. Полдня.
3. **neuralforecast**: лёгкие сети с полным набором сопутствующих рядов —
   **BiTCN, NHITS, NBEATSx, TSMixerx, TiDE, MLP, KAN** (`futr_exog_list`
   = закачка, `hist_exog_list` = давления, `stat_exog_list` = статика и
   блок); `cross_validation(n_windows, h=6)` — наша схема одним вызовом;
   квантильные лоссы (MQLoss и др.) и конформные интервалы
   (`PredictionIntervals`). 1–2 дня с автотюнингом на a100.

**Ловушки из исходников:** TimeXer — вопреки названию статьи, известные
наперёд ряды в реализации Nixtla не поддержаны; DeepAR — без
исторических; PatchTST и iTransformer — вообще без сопутствующих рядов;
TFT — самый прожорливый, на 3300 наблюдениях переобучится первым;
CrostonClassic — модель прерывистого спроса, не для дебитов.
hierarchicalforecast (согласование скважина→блок→месторождение) — только
если нужны согласованные суммы по уровням; на скважинном уровне выигрыш
0–3 %. TimeGPT — платный, дублирует TiRex-2.

## 4. Итоговое суждение и план этапа 5

Внешний обзор верен по духу и неточен по фактам: его первое место
(Toto 2.0) на нашей задаче дисквалифицируется отсутствием сопутствующих
рядов, реальная интрига — **TiRex-2 против Chronos-2**, и оба — против
физической связки CRM × Джентил. Урок трёх полигонов остаётся в силе:
положение в таблицах лидеров вторично, решают поддержка управляющих
сопутствующих рядов и характер данных.

План (по убыванию отношения польза/затраты, суммарно ~3–4 дня):

| № | Шаг | Ожидание | Затраты |
|---|-----|----------|---------|
| 1 | Chronos-2 на трёх полигонах (закачка как известный план, наш бэктест) | конкурент/партнёр TiRex-2; № 1 на fev-bench | ~1 день |
| 2 | LightGBM с закачкой, блоками и статикой (mlforecast или darts) | сильный обучаемый эталон, 40–50 % шанс на 7–9 % | ~1 день |
| 3 | Классика: AutoARIMA (с закачкой), AutoETS, AutoTheta | легитимная нижняя планка для отчёта | ~полдня |
| 4 | Toto-2.0-313m | верхняя планка «без сопутствующих рядов», количественная цена ковариат | ~полдня |
| 5 | Ансамбль: CRM × Джентил + лучшая предобученная + LightGBM | единственный реальный кандидат уйти ниже 6.4 % на нашем поле | ~полдня |
| 6 | Расширение скользящей проверки до 10–20 срезов (darts-контур) | надёжнее выбор моделей и калибровка интервалов | ~полдня |
